I was inspired by the HTML5 canvas feature and some projects implemented using canvas, so I made this signature board based on the references supported by HTML5 canvas. Users can drag the mouse to sign on the board, click to change the color which is randomly generated, clear the board, and save it to your folder. In order to make some fancy effects on the stroke, I manipulated the parameters in quadraticCurveTo() method. Check it out!

Knitter is a chrome extension provided for users to quickly capture and store the web content to the Dropbox while browsing. It's not enough just to "Save" encountered information. We keep the information in ways so that we can find it again later when and where we'll need it. We make connections between new information and the corresponding folders in the Dropbox for which that information is relevant.In this project, UI is implemented using HTML5, CSS3, Bootstrap. The toggle sidebar is constructed using Javascript and the connection between the content and dropbox is implemented by the API -- ItemMirror developed by UW ischool team. We also work with Dropbox API for Dropbox identification.

The water drops animation feature is implemented by CSS3. My idea comes from a Chinese proverb -- Ji Shao Cheng Duo which means many a little makes a micle in English. Therefore, we can see there is a water vat that collects all drops and finally a vast amount of drops fills up the vat. To make it more interesting, I also added a "dancing" 2015 at the top. Check it out!

This is a simple drag and drop feature which I created as a small part of my project. Here is a 2 x 2 grid of input elements where inputs on the left side are draggable and the inputs on the right side are droppable. The inputs will be vertically labled A and B, respectively. When a user drags an input that contains a value the value should carry over to the droppable input into which it is dropped. Users should be able to drag the left A to the right A, but not A to B, or vice versa. jQuery is used and also the Jasemine framework is applied for the unit tests. Check it out.

I, with other 4 classmates set up this e-commerce website when I was in the last year of undergraduate. And that is about my very first step of my entrepreneurship. Also I was in charge of some parts of database management. Check it out!

Car Park System is one of my projects during my undergraduate study and I built it using Java. In this project, I realized four basic functional modules: Entrance which mainly shows the current parking lots status to users, allows staffs log in and generate tickets for customer; Payment station which calculates parking fees, process the payments and generate receipts for payer; Exit which verifies the paid tickets and allows users exit; Main control and staff management which allow admin to log in, check the car park reports and manage staff accounts.Also, Java Swing APIs are used to implement the user interfaces and XML Reader Interfaces are implemented to set and query data about customer, staffs, tickets and current status of the parking lots. Check it Out!

Moolly is a web and mobile application designed for smartphone users who suffers or may suffer from mood and anxiety disorders. Moolly observes patterns of expression and behavior of a smartphone user, based on the users physical activities and self-reported mood information, and provides valuable insight about the user's response to medication and therapy, with a goal to help the user recover sooner or feel better. Check it out!

I visualzed the disasters distribution around the world in 2008 by highmap APIs. The data comes from US statistics 2008. The world map gives the overview of the distribution of a variety of disasters. In this map, different types of disaster are norminal data, so I used different colors to classify them. The tooltip adds details information (location, the number of people killed, etc.) to a certain type of disaster. Viewers can zoom in or out the map, and also download the map. Check it out!
